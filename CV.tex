% --- 第一个项目 ---
\textbf{MiniOneRec - 端到端生成式推荐系统框架 (个人核心 / 开源项目)} \hfill 核心算法开发 \titlebreak
\textbf{项目背景}：针对传统推荐系统冷启动困难及多模态语义融合效果差的问题，突破直接引入 LLM 存在庞大词表导致生成效率低下和推荐“幻觉”的痛点。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{表征构建 (语义量化)}：采用文本编码器提取多模态特征，应用多级 RQ-VAE / Constrained RQ-Kmeans 算法对百万级候选物品进行量化，构建定长、均匀分布的层级化语义 ID (SID)，有效缓解特征稀疏性；引入 Accelerate 实现多 GPU 并行推理，\textbf{特征提取效率提升 300\%}。
    \item \textbf{指令对齐微调 (SFT)}：将用户时间序列交互历史转化为 Token，利用自回归大模型（如 Qwen2.5-base）进行序列推荐微调；创新引入语言对齐目标与价值感知微调 (VAFT)，使模型兼具外部世界知识并与离散 ID 空间高效对齐。
    \item \textbf{策略对齐强化 (RL \& GRPO)}：弃用高显存占用的 PPO Critic 网络，\textbf{设计基于 GRPO 的轻量级强化学习}。通过组内相对 Reward 稳定梯度，并融合二元正确率、排序置信度 (Rank-aware 惩罚) 与 CF 分数构建混合奖励信号，打破历史曝光偏差。
    \item \textbf{受限解码与项目成果}：开发定制化 LogitProcessor，在 Beam Search 阶段实施前缀树 (Trie) 约束的受限解码。确保输出 SID 100\% 唯一合法，\textbf{推理侧无效生成 (CC 指标) 降至 0}；最终在 Amazon 数据集上，相比传统基线，\textbf{HR@10 提升 15.6\%，NDCG@10 提升 18.2\%}。
\end{itemize}

% --- 第二个项目 ---
\textbf{面向 OTTO 电商平台的大规模用户行为预测} \hfill 算法与特征工程 \titlebreak
\textbf{项目背景}：在包含 1.64 亿条交互日志、1100 万+ 匿名会话及 180 万+ 商品的超大规模数据集上，精准预测用户的点击、加购、下单行为，并为每位用户输出 Top-20 推荐列表。
\begin{itemize}[nosep, topsep=\intraProjectListTopSep]
    \item \textbf{多路召回设计}：构建 20 种共访矩阵，创新性引入指数时间衰减权重 $w=(1/2)^{|\Delta t|/3600}$，精准捕获时间敏感特征，为每位用户高效召回 100 个高质量候选商品。
    \item \textbf{特征工程与树模型排序}：深度构建 100+ 维特征，涵盖共访矩阵分数、商品/用户侧多维度统计及多时间窗口聚合；分别训练 Click/Cart/Order 三个 \textbf{XGBoost} 二分类模型，采用 40\% 负样本下采样与严格时间切分有效避免数据泄漏。
    \item \textbf{GPU 工程优化落地}：全流程基于 \textbf{RAPIDS cuDF} 实现 GPU 极速运算，使\textbf{共访矩阵计算获得 30 倍性能飞跃}；采用分块计算结合 Parquet 列式存储及主动内存回收策略，成功跑通端到端亿级数据流水线。最终将 XGBoost 与 CatBoost 输出概率线性加权融合，在 Kaggle 线上 Leaderboard \textbf{斩获 0.604 的高分表现}。
\end{itemize}
